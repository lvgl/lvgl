## Overview

Buttons have no new features compared to the [base widget](/widgets/base_widget).
They are useful for semantic purposes and have slightly different default settings.

Buttons, by default, differ from the Base Widget in the following ways:

- Not scrollable
- Added to the default group
- Default height and width set to <ApiLink name="LV_SIZE_CONTENT" />

## Parts and Styles

- <ApiLink name="LV_PART_MAIN" /> The background of the button; uses the [typical
   background style properties](/common-widget-features/styles/overview).

## Usage

There are no new features compared to [Base Widget](/widgets/base_widget).

### Basics

A button is a clickable Base Widget. The default `LV_SIZE_CONTENT` width and
height let it wrap tightly around its label child; setting an explicit `width`
or `height` overrides this.

### States

Buttons inherit base-widget states. The two most relevant for buttons are
<ApiLink name="LV_STATE_CHECKED" /> (combined with the
<ApiLink name="LV_OBJ_FLAG_CHECKABLE" /> flag so a click toggles the state)
and <ApiLink name="LV_STATE_DISABLED" />, which mutes the button and stops
input events.

### Styling

<LvglXMLExample name="lv_example_button_styling" path="widgets/button/button_styling/lv_example_button_styling" />

A button only has <ApiLink name="LV_PART_MAIN" />, but it picks up the
full base-widget style stack: `bg_color`, `radius`, `border_*`,
`outline_*`, `shadow_*`, `pad_*`, and `text_color`. The interesting
piece is state-aware styling: declare separate `<style>` blocks and
attach them with `selector="pressed"`, `selector="checked"`,
`selector="focused"`, or `selector="disabled"` so the rendered button
changes its look as the user interacts with it. Local `style_*` props
on the `<lv_button>` tag are a shortcut when you only need a single
state.

## Events

- <ApiLink name="LV_EVENT_VALUE_CHANGED" /> when the <ApiLink name="LV_OBJ_FLAG_CHECKABLE" /> flag is
   enabled and the Widget is clicked. The event happens on transition
   to/from the checked state.
