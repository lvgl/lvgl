## Overview

ESP32-P4 includes a 2D Direct Memory Access peripheral (DMA2D) that can speed up
2D block transfers and selected color conversions.

In LVGL, the Espressif DMA2D draw unit can be used for:

- `IMAGE` and `LAYER` draw tasks that are simple blit operations.
- RGB format conversion between `RGB565` and `RGB888`.

The unit is designed to coexist with software rendering and PPA:

- DMA2D targets copy/convert paths.
- PPA targets fill/blend paths.
- SW renderer remains the fallback for unsupported cases.

More information about PPA can be found in
[PPA (Pixel Processing Accelerator) Support](/integration/chip_vendors/espressif/hardware_accelerator_ppa).

## Enabling DMA2D draw unit in LVGL

Enable DMA2D draw unit in your ESP-IDF project `sdkconfig.defaults`:

```c
CONFIG_LV_USE_ESP_DMA2D=y
CONFIG_LV_ESP_DMA2D_BURST_LENGTH=64
CONFIG_LV_DRAW_BUF_ALIGN=64
```

After rebuilding, LVGL will automatically evaluate tasks and dispatch compatible ones to DMA2D.

## Supported use cases

Current Espressif DMA2D draw unit supports:

- `LV_DRAW_TASK_TYPE_IMAGE` and `LV_DRAW_TASK_TYPE_LAYER`
- no rotation/scale/skew
- no recolor
- no tiling
- no bitmap mask
- blend mode `NORMAL` with full opacity
- color formats:
  - source: `RGB565`, `RGB888`
  - destination: `RGB565`, `RGB888`
  - conversion: `RGB565 <-> RGB888`

Unsupported operations automatically fall back to the software draw unit.

## Notes about display DMA2D

Besides the LVGL draw unit, some display pipelines may also use DMA2D at driver level
for frame-buffer movement/tear avoidance.

That driver-level DMA2D path is independent from LVGL draw-unit dispatching, and both can be
enabled in the same project as long as memory bandwidth is sufficient for the use case.
